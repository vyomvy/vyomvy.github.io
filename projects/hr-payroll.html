HR & payroll SQL analysis

A self-designed relational database, built and queried from scratch to practice the SQL analysts actually use on the job.

Source code ↗
The problem

Most beginner SQL practice stops at SELECT and basic joins. This project was built to go further — designing the schema itself, then writing the kind of queries that show up in real analyst work.

The approach
01
Design the schema

Built a 3-table relational database — employees, departments, orders — from scratch.

02
Write real queries

Joins, self-joins, correlated subqueries, LAG() window functions, and running totals.
