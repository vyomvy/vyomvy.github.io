Pipeline 01 / Food delivery

Food delivery business analysis

A full Python → SQL → Power BI pipeline built to find where a food delivery operation was quietly losing money.

Live dashboard ↗ Source code ↗
The problem

A food delivery business had 1,500+ orders across 10 areas and no clear read on where refunds were actually coming from. Revenue looked fine at a glance — the question was whether it was fine everywhere, or whether some segments were quietly bleeding money.

The pipeline
01 / PYTHON
Clean the raw data

Standardized order records, handled missing values, and prepped the dataset for querying.

02 / SQL
Run the analysis

Used CTEs and window functions in MySQL to segment orders by distance band and order type.

03 / POWER BI
Build the dashboard

Turned the query output into an interactive dashboard for ongoing monitoring.

Key findings
17%

Refund rate for orders placed beyond 9km — even though that distance band had the highest order value.

#1

First-time orders were the single highest-refund segment, pointing to a likely discount-abuse pattern.

10

Delivery areas segmented and compared to isolate where the refund pattern was concentrated.
